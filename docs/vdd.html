Manual de Referência

[VDD]

Esta tag permite definir regras de validação para edição de campos (na tela de parâmetros definida por [PAR] ou no grid, quando utilizada a tag [EDT]).

Variável [$n$] refere-se à coluna n ([EDT]) em edição.

Sintaxe

+> ID : WHERE : TIPO : MENSAGEM

O ID deve ser igual ao ID do [EDT] em caso de utilização no [EDT]. Caso for utilizado no [PAR], o ID do VDD deve ser informado no parâmetro a ser validado (vide sintaxe padrão dos parâmetros).

O WHERE é a condição a ser utilizada para a validação. O elemento (parâmetro ou EDT) a ser validado é válido quando a condição retornar FALSO.

O TIPO pode ser E ou W, onde W é warning (alerta) e E é error (erro). O tipo W de um elemento inválido mostrará o alerta, porém permitirá prosseguir. O tipo E de um elemento inválido mostrará um erro ao tentar dar sequência à operação e não permitirá prosseguir.

A MENSAGEM é o que será exibido quando o elemento for inválido e o usuário tentar prosseguir com a operação. Em caso de erro no banco, a mensagem retornada será do erro do banco, e não do VDD

Exemplo de código

[VDD]
+> 1 : ‘X[$EDT$]’ = ‘X’ :E : Campo deve ser preenchido
+> 1 : length(‘X[$EDT$]’ )<11 :E : Campo deve ter, pelo menos, 10 caracteres
